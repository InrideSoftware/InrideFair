# Политика безопасности

## Поддерживаемые версии

| Версия | Поддержка          |
|--------|--------------------|
| 3.2.x  | ✅ Активная поддержка |
| 3.1.x  | ⚠️ Ограниченная поддержка |
| < 3.1  | ❌ Не поддерживается |

## Сообщение об уязвимости

Если вы обнаружили уязвимость безопасности, пожалуйста, **не создавайте публичный issue**. Вместо этого:

### 📧 Отправьте отчёт на email

**security@inride.software**

### 📋 Что включить в отчёт

1. **Описание уязвимости** — подробное описание проблемы
2. **Шаги воспроизведения** — как воспроизвести уязвимость
3. **Версия ПО** — версия Inride Fair и ОС
4. **Потенциальный ущерб** — как уязвимость может быть использована
5. **Рекомендации** — возможные способы исправления (если есть)

### ⏱️ Время ответа

- **Подтверждение получения**: в течение 48 часов
- **Оценка критичности**: в течение 7 дней
- **Исправление**: зависит от критичности (1-30 дней)

## Типы уязвимостей

### Критичные
- Удалённое выполнение кода (RCE)
- Обход проверки сигнатур
- Раскрытие чувствительных данных
- Повышение привилегий

### Средние
- Ложные срабатывания
- Проблемы с валидацией входных данных
- Уязвимости в зависимостях

### Низкие
- Косметические проблемы
- Небольшие недочёты в логировании

## Процесс исправления

1. **Получение отчёта** → Анализ и подтверждение
2. **Оценка риска** → Определение критичности
3. **Разработка патча** → Исправление уязвимости
4. **Тестирование** → Проверка исправления
5. **Релиз** → Публикация обновлённой версии
6. **Disclosure** → Публичное описание (после релиза)

## Безопасное использование

### Рекомендации пользователям

1. **Запуск от администратора** — требуется для полной проверки системы
2. **Проверка хеша** — сверяйте хеш скачанного файла с официальным
3. **Обновления** — всегда используйте последнюю версию
4. **Антивирус** — добавьте Inride Fair в исключения (возможны ложные срабатывания)

### Что делает программа

✅ **Разрешено:**
- Сканирование файлов и процессов
- Чтение истории браузеров
- Проверка реестра
- Создание локальных отчётов

❌ **Запрещено:**
- Удаление файлов без согласия
- Отправка данных в интернет
- Изменение системных файлов
- Постоянный сбор данных
